# 这个文件用来给作者打草稿

OK
这个菜单的设计目的如下：
1. 提供快速拓展接口，让以后开发一些小程序可以快速运行
2. 打造通用框架，方便快速调试不同硬件
3. 让自己开心

所以根据1. ， 所有函数/枚举命名 不用追求严谨，只要简洁易读就行了
根据2. ，adapter里一定要有个抽象层

###  层级 -> 职责  分配

0_config   提供一些底层的计算函数/宏/全局配置
1_driver   提供菜单必须的一些驱动，比如键盘
2_adapter  调用输入设备驱动，把输入信号转化成事件
3_core     提供核心菜单项的逻辑函数和类
4_display  提供显示逻辑，把菜单类转换为图像
5_APIs     提供高级函数包装

#### 细分功能
- 1_driver
 - keyboard :键盘扫描函数
  - 0_按键枚举定义
  - 1_键盘扫描

- 2_adapter
 - keyboardEvent :键盘输入-事件转换器
  - 0_事件类型定义
  - 1_定义信号种类和判断逻辑
  - 2_定时扫描键盘并判断事件，结果放入队列

- 3_core
 - 0_menu :定义基本的菜单类和行为 (事件队列触发)
 - 1_types:定义菜单类的子类和行为
