# ESP32S3-SSD1306-menu-v2
- 第二版菜单框架
- 本项目基于 ESP32-S3，可在作者设计的定制盾板上直接运行，无需额外调试；同时也兼容其他 ESP32-S3 开发板，仅需简单配置引脚映射/按键扫描即可适配。
- 本项目为platformIO项目，基于Arduino框架
- 在VScode运行时可以下载wokwi扩展来仿真运行

## 快速开始
下列内容教你如何快速应用该菜单

0. 主要功能
- 该菜单用于简单项目，可以在一个简单的图像化界面中选择并调用用户自己写的函数(支持lambda和void()类)
- 该项目使用简单，适合新手
- 同时项目结构清晰，可以拿来学习交流
1. 安装
该库没有上传到arduino之类的平台，所以需要用户自行下载并导入
具体步骤如下
- 在安装该库之前先装好```FreeRTOS```库 和 ```u8g2```库
- 下载项目文件夹中的 ```lib/menu_v2.1``` 文件夹
- 将该文件夹整个放到你的项目库中
- 在你的库中#include ".../menu_v2.1/menuv2.h"
2. 配置
主要是初始化u8g2和设置键盘引脚
目前只做了独立键盘的扫描，所以用其他输入需要自行更改代码
如果有想要的功能可以交个issue
- 详情见menuv2.1/config.h，里面有注释
3. 使用
- 创建菜单
```cpp
menu Your_Menu_Name(
    "name",{
        {"option_01",Function},  // 创建一个“选中后执行Function(void)一次”的选项 在 Your_Menu_Name中
        {"option_02",[](){Serial.println("原神牛逼");}}, // 支持lambda表达式，可借此调用带参函数
        {"option+03",newMenu(    // 内置宏，创建一个"新建并进入子菜单"的函数
            "name_2",{
                {"option_03"},   // 支持空选项
                {"option_04"},
            }
        )},           
    }
)
/* 
解释：menu 是一个类 ，算是一种类型
Your_Menu_Name是类的一个对象，后面括号里是创建对象的参数
(这是基本的C++特性，但是我看好多人不会，你们快去学)
这个参数也算是一种函数的参数，不懂理解成调用函数就行了

创建menu类型对象
参数有两个
1. name          决定创建菜单后显示的名字,填字符串
2. itemTable[]   决定菜单内容,填menuItem类(选项类)数组                             (可表示为{元素1,元素2, ...})
所以 括号里的外层大括号 代表创建一个数组
里面的其他大括号就是这个数组里的元素(menuItem类)，此类元素如下
2.1. name;            // 选项显示的名字,填字符串
2.2. run;             // 行为函数，选中调用这个     (高级函数指针)                  (std::function<void()>类)
整理一下
大概是下面这样
*/
menu Menu(
    /*显示名字*/"Menu",
    /*选项数组(NoName)*/{
        /*选项1*/{"name",func}, // func只能是void(void)函数指针和lambda表达式
        ...                     // 菜单内置的newMenu会创建lambda，不用lambda没法传参数
    }
)
```
这样你就创建了一个菜单
- 进入主菜单
```cpp
menuInit(Your_Menu_Name); //菜单系统初始化
/*
这个函数会初始化菜单相关的任务，并将参数作为主菜单启动
接下来你啥都不用干，系统会创建两个线程
一个采集按键，一个显示
至此，你已进入菜单
*/
```
